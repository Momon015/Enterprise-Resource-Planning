{% comment %} action= IS LOAD-BEARING, DO NOT REMOVE IT. This form now sits inside the boosted #waste-results region. Vendored htmx 2.0.3 boosts a form like this: o = getAttribute(form, "action"); if (verb === "get" && o.includes("?")) { ... } With no action attribute o is null, so o.includes() throws a TypeError — inside htmx's DOM-processing pass, which aborts. Every element after this form (the whole table and its pagination) then never gets wired up and falls back to native navigation: a full page reload on every filter. Silently — the only trace is one console error. htmx 2.0.4+ guards the null (path = location.href). Until we upgrade, every boosted form in this app MUST carry an explicit action. {% endcomment %}
