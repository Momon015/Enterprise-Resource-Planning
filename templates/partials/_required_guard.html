{% comment %} Client-side required-field guard for the htmx modal forms. WHY THIS EXISTS: these forms carry `novalidate`, so the browser's own required-field bubble is off. Without a guard an empty submit POSTs, the view finds the form invalid and re-renders the modal, and htmx swaps the whole panel — the modal visibly "refreshes", scroll position resets, and on any form with a file input THE CHOSEN FILE IS LOST (a browser will not re-attach a file to a re-rendered input). Catching it here means the form never round-trips for something the page already knows. `novalidate` stays ON PURPOSE — do not "simplify" this away by dropping it and letting the browser validate. These forms hide fields conditionally (a product linked to a material renders prepared_quantity as a hidden input), and native validation on a hidden required control aborts the submit with an unfocusable-control console error and NO visible message — a dead Save button with no explanation. USAGE: mark each genuinely-required visible input with `data-req`, then include this once inside the form: {% include 'partials/_required_guard.html' with form_id='materialForm' %} Required sets differ per form (supplier = name; material = name/price/quantity/unit/ piece_per_unit), which is exactly why the fields declare themselves instead of this snippet hardcoding a list. {% endcomment %}
